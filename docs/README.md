## 文档元数据规范（建议）
- 头部统一包含以下信息，便于检索与维护：
  - 版本：例如 `v0.1 初稿`
  - 状态：`草案 | 进行中 | 已发布 | 废弃`
  - 更新日期：`YYYY-MM-DD`
  - 责任人/协作人：`@owner`（可选）
  - 关联文档：指向相关设计/API/测试/迁移/故障报告等

示例（非 YAML，仅行内约定）：
```
版本：v0.1 初稿  
状态：进行中  
更新日期：2025-12-28  
责任人：@owner
```

## 维护约定与命名
- 文件命名使用清晰中文标题，必要时加入日期标识（如 `-2025-12-26`）以区分报告。
- 目录归类以主题为准：需求与设计、实施与配置、测试与质量、故障与报告、API、里程碑、状态更新、技术迁移。
- 每次结构性调整后更新本索引，确保新增/移动的文档可在此处快速定位。
- 文档末尾建议加入“相关文档”章节，交叉链接到：
  - `API/API规范.md`
  - `实施与配置/环境配置.md`
  - `测试与质量/测试计划.md`
  - `技术迁移/从SpringDataJPA迁移到MyBatis-Plus.md`
  - 相关故障报告与里程碑文档
